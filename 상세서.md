# Live Meeting AI Demo 상세서

이 문서는 실시간 음성 처리 기반 회의록 요약 웹 데모의 구현 기준이다. UI/UX, 화면 구조, 상태값, 반응형 규칙, 접근성, 예외 처리를 개발과 디자인에서 그대로 적용할 수 있는 수준으로 정의한다.

> **상태**: PLANNED - Implementation Pending  
> **전제**: 내부 AI 모델/API는 이미 준비되어 있으나, 정확한 API 계약서가 제공되지 않았다. 이 문서의 API 경로와 이벤트명은 웹 서비스 구현을 위한 백엔드 어댑터 기준 제안이다. 실제 내부 API 명세가 확정되면 `10. API 어댑터 계약`만 교체한다.

## 1. 제품 범위

### 1.1 제품명

`Live Meeting AI Demo`

### 1.2 목적

브라우저 마이크 입력을 WebSocket 기반으로 백엔드에 전송하고, 내부 AI 음성 처리 API와 연동해 실시간 자막, 중간 요약, 최종 회의록, 액션 아이템을 제공하는 데모 웹 서비스.

### 1.3 MVP 포함 범위

| 구분        | 포함 기능                                                 | 구현 기준                                                             |
| ----------- | --------------------------------------------------------- | --------------------------------------------------------------------- |
| 오디오 입력 | 브라우저 마이크 권한 요청, 입력 장치 선택, 음량 레벨 표시 | `MediaDevices.getUserMedia`, `MediaRecorder` 또는 `AudioWorklet` 사용 |
| 실시간 전송 | 오디오 chunk를 WebSocket으로 백엔드 전송                  | 기본 chunk 간격 250ms, 설정 가능 범위 100-1000ms                      |
| 실시간 자막 | partial/final transcript 표시                             | partial은 회색/기울임, final은 일반 텍스트                            |
| 실시간 요약 | 30초 또는 60초 단위 요약 갱신                             | 기본 60초                                                             |
| 결과 검토   | 최종 요약, 결정사항, 액션 아이템, 전체 자막 확인          | 회의 종료 후 결과 화면 이동                                           |
| 내보내기    | `.md`, `.txt`, `.json` 다운로드                           | 브라우저 다운로드                                                     |
| 상태 표시   | 마이크, WebSocket, AI API, 지연시간, 저장 상태 표시       | 상단 상태 바와 토스트 사용                                            |

### 1.4 MVP 제외 범위

| 제외 항목                  | 사유                                |
| -------------------------- | ----------------------------------- |
| 사용자 로그인/조직 관리    | 데모 목적상 불필요                  |
| 장기 보관용 회의 히스토리  | 개인정보/스토리지 정책 확정 전 제외 |
| 다중 사용자 동시 회의 참여 | 1인 입력 데모에 집중                |
| 화자 분리 정확도 보장      | 내부 모델 성능 정보 없음            |
| 결제/권한/관리자 화면      | 프로젝트 어필용 데모 범위 초과      |

### 1.5 성공 기준

| 항목         | 기준                                                         |
| ------------ | ------------------------------------------------------------ |
| 시작 난이도  | 첫 화면에서 2회 이하 클릭으로 녹음 시작                      |
| 첫 자막 표시 | 사용자가 발화한 뒤 목표 3초 이내 첫 partial 표시             |
| 상태 인지    | 연결 끊김, 권한 거부, API 지연을 화면에서 1초 이내 식별 가능 |
| 반응형       | 360px, 768px, 1366px, 1920px 폭에서 가로 스크롤 없음         |
| 접근성       | WCAG 2.1 AA 기준 준수                                        |
| 데모 완결성  | 녹음 시작부터 최종 회의록 다운로드까지 단일 흐름으로 완료    |

## 3. 사용자 흐름

### 3.1 정상 흐름

1. 사용자가 `/` 진입.
2. 앱이 브라우저 지원 여부를 확인한다.
3. 사용자가 `마이크 확인`을 클릭한다.
4. 브라우저 마이크 권한 요청이 열린다.
5. 권한 허용 후 입력 장치와 음량 레벨이 표시된다.
6. 사용자가 회의 제목, 언어, 요약 주기를 확인한다.
7. 사용자가 `실시간 회의 시작`을 클릭한다.
8. 백엔드가 세션을 생성하고 `/session/:sessionId`로 이동한다.
9. WebSocket 연결 후 오디오 chunk 전송을 시작한다.
10. 서버가 `transcript.partial` 이벤트를 보내면 실시간 자막에 임시 문장을 표시한다.
11. 서버가 `transcript.final` 이벤트를 보내면 임시 문장을 확정 문장으로 교체한다.
12. 서버가 `summary.snapshot` 이벤트를 보내면 오른쪽 요약 영역을 갱신한다.
13. 사용자가 `종료`를 클릭한다.
14. 앱이 서버에 `session.stop`을 보내고 최종 결과 생성을 대기한다.
15. 최종 결과 수신 후 `/session/:sessionId/result`로 이동한다.
16. 사용자가 결과를 검토하고 `.md`, `.txt`, `.json` 중 하나로 다운로드한다.

### 3.2 권한 거부 흐름

1. 사용자가 마이크 권한을 거부한다.
2. 앱 상태를 `permission_denied`로 변경한다.
3. 첫 화면 상단에 오류 배너를 표시한다.
4. `브라우저 설정에서 마이크 권한을 허용한 뒤 다시 시도하세요.` 문구를 표시한다.
5. `다시 확인` 버튼을 제공한다.
6. 브라우저 정책상 재요청이 불가능하면 `설정 안내 보기` 버튼으로 권한 안내 모달을 연다.

### 3.3 연결 불안정 흐름

1. WebSocket heartbeat가 5초 동안 응답하지 않는다.
2. 상태를 `reconnecting`으로 변경하고 상단 배너를 표시한다.
3. 전송 큐에 최대 10초 분량의 오디오를 보관한다.
4. 1초, 2초, 4초 간격으로 최대 3회 재연결한다.
5. 재연결 성공 시 큐를 순차 전송하고 `연결 복구됨` 토스트를 표시한다.
6. 실패 시 상태를 `failed`로 변경하고 녹음을 자동 중지한다.

### 3.4 결과 없음 흐름

1. 사용자가 10초 이상 무음 상태로 녹음을 종료한다.
2. 결과 화면에 빈 상태를 표시한다.
3. 최종 요약 영역에는 `분석할 음성이 충분하지 않습니다.`를 표시한다.
4. 다운로드 버튼은 비활성화한다.
5. `새 회의 시작` 버튼을 제공한다.

## 4. 화면별 구조

### 4.1 공통 앱 셸

| 요소        | 위치                          | 크기/규칙                                           |
| ----------- | ----------------------------- | --------------------------------------------------- |
| 상단 바     | 화면 최상단 fixed 또는 sticky | 높이 56px, 좌우 padding 16px 모바일 / 24px 데스크톱 |
| 메인 영역   | 상단 바 아래                  | `min-height: calc(100dvh - 56px)`                   |

상단 바 배경은 `--color-surface`, 하단 border는 `1px solid --color-border-subtle`을 사용한다.

### 4.2 세션 설정 화면

#### 4.2.1 목적

사용자가 마이크 권한과 입력 장치를 확인하고, 실시간 회의 세션을 시작한다.

#### 4.2.2 데스크톱 레이아웃

| 영역          | 그리드       | 규격                                             |
| ------------- | ------------ | ------------------------------------------------ |
| 메인 컨테이너 | 12 columns   | max-width 1120px, margin auto, padding 32px 24px |
| 설정 폼       | columns 1-7  | 폭 100%, gap 20px                                |
| 상태 패널     | columns 8-12 | sticky top 80px, gap 16px                        |

#### 4.2.3 모바일 레이아웃

| 영역          | 규격                                   |
| ------------- | -------------------------------------- |
| 메인 컨테이너 | padding 20px 16px                      |
| 설정 폼       | 단일 컬럼                              |
| 상태 패널     | 폼 아래 배치                           |
| 시작 버튼     | 화면 하단 sticky, 높이 56px, 좌우 16px |

#### 4.2.4 콘텐츠 구조

| 순서 | 컴포넌트                    | 필수 여부 | 규칙                                |
| ---- | --------------------------- | --------- | ----------------------------------- |
| 1    | 화면 제목                   | 필수      | `실시간 회의 시작`                  |
| 2    | 회의 제목 입력              | 선택      | 기본값 `새 회의 - YYYY.MM.DD HH:mm` |
| 3    | 언어 선택                   | 필수      | 기본값 `한국어`                     |
| 4    | 입력 장치 선택              | 필수      | 권한 허용 후 장치 목록 표시         |
| 5    | 요약 주기 segmented control | 필수      | `30초`, `60초`, `수동`              |
| 6    | 마이크 테스트 패널          | 필수      | 음량 meter, sample rate, 권한 상태  |
| 7    | API 상태 패널               | 필수      | STT, 요약, 액션 추출 API 상태       |
| 8    | 시작 버튼                   | 필수      | 권한과 API 상태가 정상일 때 활성화  |

#### 4.2.5 유효성 규칙

| 필드      | 조건                       | 오류 문구                             |

| 언어      | 지원 언어 중 하나          | `지원하지 않는 언어입니다.`           |
| 입력 장치 | 권한 허용 후 deviceId 존재 | `사용 가능한 마이크가 없습니다.`      |

#### 4.3.5 인사이트 패널

| 탭       | 목적                 | 기본 표시      |
| -------- | -------------------- | -------------- |
| 요약     | 현재까지의 회의 요약 | 기본 활성      |
| 할 일    | 액션 아이템 목록     | 건수 배지 표시 |
| 결정사항 | 확정된 결정 목록     | 건수 배지 표시 |
| 키워드   | 주요 키워드          | 최대 20개      |

요약 탭 규칙:

| 요소           | 규격                              |
| -------------- | --------------------------------- |
| 현재 요약      | 3-7개 bullet, 각 bullet 80자 이하 |

| 수동 갱신 버튼 | 요약 주기 `수동`일 때 활성        |
| 요약 생성 중   | skeleton 3줄 표시                 |

할 일 탭 규칙:

## 10. API 어댑터 계약

실제 내부 AI API의 확정 정보가 없으므로 웹 앱은 백엔드 어댑터와만 통신한다. 백엔드는 내부 API 스펙을 감싸고 프론트엔드에 안정적인 이벤트 계약을 제공한다.

### 10.0 최종 구현 구조

현재 구현은 `Supabase Edge Function + Cloud Run Node bridge` 구조를 기준으로 한다.

| 영역                  | 구현                                            |
| --------------------- | ----------------------------------------------- |
| 제품 API              | Supabase Edge Function `meeting-api`            |
| 실시간 오디오 bridge  | Cloud Run Node 서비스 `stt-bridge`              |
| 세션/결과 저장        | Supabase Postgres                               |
| 실시간 STT            | Google Cloud Speech-to-Text streaming gRPC      |
| 요약/결정/액션 아이템 | Gemini `generateContent`                        |
| 브라우저 오디오 형식  | AudioWorklet 기반 16kHz mono PCM16, 250ms chunk |

Google Speech-to-Text streaming은 gRPC 기반이므로 브라우저와 Google API를 직접 연결하지 않는다. 브라우저는 `stt-bridge` WebSocket으로 audio chunk를 보내고, bridge가 Google STT와 Gemini를 호출한 뒤 Supabase에 결과를 저장한다.

### 10.1 HTTP 엔드포인트

| Method | Path                       | 목적                            |
| ------ | -------------------------- | ------------------------------- |
| `GET`  | `/api/health`              | 백엔드, STT, 요약 API 상태 확인 |
| `POST` | `/api/sessions`            | 세션 생성                       |
| `GET`  | `/api/sessions/:id`        | 세션 메타 조회                  |
| `GET`  | `/api/sessions/:id/result` | 최종 결과 조회                  |
| `POST` | `/api/sessions/:id/export` | export payload 생성             |

### 10.2 WebSocket

| Path                      | 목적                                   |
| ------------------------- | -------------------------------------- |
| `/api/sessions/:id/audio` | 오디오 chunk 전송과 실시간 이벤트 수신 |

### 12.2 Retry 규칙

| 대상           | 횟수 | 간격       | 실패 후            |
| -------------- | ---: | ---------- | ------------------ |
| WebSocket 연결 |  3회 | 1s, 2s, 4s | `failed`           |
| AI API 요청    |  2회 | 1s, 3s     | 해당 블록 `failed` |
| Export 생성    |  2회 | 즉시, 1s   | error toast        |
| Health check   | 계속 | 10s        | 상태 badge 갱신    |

### 12.3 녹음 중 페이지 이탈

조건:

- `recording`, `paused`, `reconnecting`, `processing_final` 상태에서 브라우저 뒤로가기, 새로고침, 탭 닫기 발생.

처리:

1. `beforeunload` 경고를 표시한다.
2. 앱 내부 라우팅이면 종료 확인 모달을 표시한다.
3. 사용자가 `종료하고 나가기`를 선택하면 `session.stop`을 먼저 전송한다.
4. 3초 내 응답이 없으면 로컬 상태를 `interrupted`로 기록하고 이동한다.

### 12.4 무음 처리

## 13. 구현 완료 기준

### 13.1 기능 완료

- 마이크 권한 확인, 장치 선택, 음량 meter가 동작한다.
- 세션 생성 후 WebSocket 연결이 수립된다.
- partial/final 자막이 구분되어 표시된다.
- 요약 snapshot이 실시간으로 갱신된다.
- 종료 후 결과 화면으로 이동한다.
- `.md`, `.txt`, `.json` 다운로드가 가능하다.
- 권한 거부, 연결 실패, API timeout, 결과 없음 상태가 화면에 표시된다.

### 13.2 UI 완료

- 정의된 색상 토큰과 폰트 체계를 사용한다.
- 버튼, 입력, 모달, toast, status badge 상태가 모두 구현되어 있다.
- 360px, 768px, 1366px, 1920px 폭에서 레이아웃 깨짐이 없다.
- 텍스트가 버튼/패널 밖으로 넘치지 않는다.
- 카드 안에 카드를 중첩하지 않는다.

### 13.3 접근성 완료

- 모든 입력에 label이 있다.
- 모든 icon-only 버튼에 `aria-label`과 tooltip이 있다.
- 모달 focus trap과 focus return이 동작한다.
- 키보드만으로 녹음 시작, 일시정지, 종료, 다운로드가 가능하다.
- 오류 메시지는 `role="alert"` 또는 `aria-live`로 전달된다.
- 자동 접근성 검사와 수동 키보드 테스트를 통과한다.
